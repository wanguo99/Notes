# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

这是一个使用 Obsidian 管理的中文技术笔记库，包含约 85 个 Markdown 文件，涵盖系统配置、开发工具使用、问题记录和工作日志。

## 目录结构与组织

项目采用数字编号的目录结构，不同范围的编号代表不同类型的内容：

- **1-7**：主要技术内容
  - `1. Linux 系统/`：Ubuntu 配置、系统问题
  - `2. Windows 系统/`：Windows 配置、工具设置
  - `3. buildroot 使用/`：buildroot 工作流程和配置
  - `4. QEMU 使用/`：QEMU 虚拟化相关
  - `5. 其他环境配置/`：特定环境安装指南
  - `6. 问题记录/`：开发中遇到的问题及解决方案
  - `7. 硬件/`：硬件相关配置

- **97**：工具配置
  - `97. 工具配置/`：包含 CLAUDE、SourceInsight、vim、zsh、rsync 等配置文件

- **98**：模板
  - `98. 模板/`：日常工作模板、任务清单模板

- **99**：工作记录
  - `99. 工作记录/`：周报、非紧急待办、项目开发记录

## 文件命名约定

- 使用数字编号前缀：`1.1.1`、`98.2.1` 等
- 文件名描述性强：`1.1.1 基础配置.md`、`3.5 编译 SDK.md`
- 编号反映层级关系（章节.小节.子节）

## Obsidian Wiki-Link 语法

文件间使用 Obsidian 的 wiki-link 语法互相引用：
- `[[文件名]]` - 基本链接
- `[[文件名|显示文本]]` - 带别名的链接
- README.md 作为导航索引，使用 wiki-link 组织所有笔记

## 特殊文件

- **备忘录.md**：存储临时性的重要信息（服务器 IP、密码等敏感信息）
- **README.md**：全局导航索引，列出所有主要笔记的 wiki-link

## 内容维护原则

1. **保持链接一致性**：修改文件名时需同步更新 README.md 和所有引用该文件的 wiki-link
2. **遵循编号体系**：新增内容时按照现有编号规则分类
3. **模板使用**：日常工作记录使用 `98. 模板/` 中的模板格式
4. **敏感信息**：`备忘录.md` 中的服务器凭证等信息不应提交到 git（但当前项目中已存在）
