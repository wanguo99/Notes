# 2.3 配置 VScode ssh-key

在 VSCode 中配置 Ubuntu 服务器的 SSH 密钥，可以实现安全、免密的远程连接。以下是详细的操作步骤：

### 第一步：生成 SSH 密钥对（本地操作）

在你的本地电脑（Windows/Mac/Linux）打开终端或命令行工具，运行以下命令生成密钥：

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

- **说明**：推荐使用 `ed25519` 算法，比传统的 RSA 更安全且生成速度更快。
- **操作**：执行后连续按 3 次回车（使用默认路径且不设置密码短语）即可。

### 第二步：将公钥部署到 Ubuntu 服务器

**方法一：使用快捷命令（推荐）** 如果你的本地环境支持 `ssh-copy-id`，可以直接运行：

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub username@服务器IP地址
```

**方法二：手动复制部署** 如果方法一不可用，可以先查看公钥内容并手动粘贴到服务器：

1. 复制公钥内容：`cat ~/.ssh/id_ed25519.pub`
2. 登录服务器：`ssh username@服务器 IP 地址`
3. 创建目录并写入公钥：

```bash
mkdir -p ~/.ssh
echo "这里粘贴你的公钥内容" >> ~/.ssh/authorized_keys
```

### 第三步：配置服务器权限（关键）

为了防止因权限问题导致连接失败，必须在 Ubuntu 服务器上执行以下命令设置正确的权限：

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

### 第四步：配置 VSCode 的 SSH Config

1. 在 VSCode 中安装微软官方的 **Remote - SSH** 插件。
2. 按 `Ctrl+Shift+P` 打开命令面板，输入并选择 `Remote-SSH: Open Configuration File`，选择你的 SSH 配置文件（如 `C:\Users\用户名\.ssh\config`）。
3. 添加以下配置以精准指定密钥路径，这能大幅提升 VSCode 的连接速度：

```text
Host my-ubuntu-server
    HostName 你的服务器IP
    User 你的用户名
    IdentityFile ~/.ssh/id_ed25519
    IdentitiesOnly yes
```

### 第五步：测试免密登录

配置完成后，按 `Ctrl+Shift+P` 选择 `Remote-SSH: Connect to Host`，点击你刚刚配置的 `my-ubuntu-server`。如果一切正常，VSCode 将会直接连接到服务器，无需再输入密码。

**排错提示**：如果在连接时依然提示需要输入密码或出现“Permission denied”，请检查 Ubuntu 服务器上 `.ssh` 目录及 `authorized_keys` 文件的权限是否严格为 700 和 600，或者检查 `/etc/ssh/sshd_config` 文件中 `PubkeyAuthentication yes` 是否已开启。
