Git 中文乱码问题通常是因为 Git 默认编码与终端编码不统一，或者 Git 默认对非 ASCII 字符（如中文路径）进行了转义。要彻底解决这个问题，可以通过以下几步进行全局配置：

### 1. 核心配置：一键修复 Git 乱码（推荐）

在 Git Bash 或命令行中执行以下三条全局配置命令，这能解决 90% 以上的中文乱码问题：

- **禁止转义中文文件名**（解决 `git status` 显示类似 `\344\270\255\346\226\207` 的问题）：
    
    ```bash
    git config --global core.quotepath false
    ```
    
- **设置日志输出编码为 UTF-8**（解决 `git log` 提交信息乱码）：
    
    ```bash
    git config --global i18n.logoutputencoding utf-8
    ```
    
- **设置提交信息编码为 UTF-8**（统一编码，避免跨平台兼容问题）：
    
    ```bash
    git config --global i18n.commitencoding utf-8
    ```
    

### 2. 终端环境适配（配置后仍乱码必做）

如果配置了上述命令后，终端依然显示乱码，通常是因为终端本身没有使用 UTF-8 编码：

- **Windows Git Bash 用户**：  
    打开 Git Bash，右键点击窗口标题栏 -> 选择 `Options` -> 左侧选择 `Text` -> 将 Locale 设为 `zh_CN`，Character set 设为 `UTF-8`。
- **VSCode 内置终端用户**：  
    VSCode 默认终端可能是 PowerShell，对中文支持较差。建议在 VSCode 设置（`Ctrl+,`）中搜索 `terminal.integrated.defaultProfile.windows`，将其修改为 `Git Bash`。

### 3. 配置验证

执行以下命令查看配置是否成功写入：

```bash
git config --global --list
```

如果你的输出中包含以下内容，说明配置已经生效：

```text
core.quotepath=false
i18n.logoutputencoding=utf-8
i18n.commitencoding=utf-8
```

### 💡 补充：关于 Git 界面提示信息的汉化

如果你不仅想解决中文乱码，还希望 Git 的**系统提示语**（如 `git status` 输出的“Changes not staged for commit”等）也变成中文，这需要通过设置系统环境变量来实现，而不是 Git 配置：  
在终端或 `~/.bashrc` 文件中添加以下环境变量：

```bash
export LANG=zh_CN.UTF-8
export LC_ALL=zh_CN.UTF-8
```

_注：Git 的界面语言依赖于系统的语言环境和是否安装了中文语言包，设置环境变量后重启终端即可生效。_
